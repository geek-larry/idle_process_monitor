# 闲置进程监控器 (Idle Process Monitor)

## 项目简介

闲置进程监控器是一个Python项目，用于监控软件进程是否闲置，并在进程闲置超过设定时间后进行优雅释放。项目通过Flask框架模拟服务端API提供进程配置信息，基于psutil和pywin32库实现进程监控和前台窗口判断。

## 功能特点

- 从服务端API获取进程配置信息，支持配置缓存，减少API调用频率
- 支持配置CPU使用率、内存占用量、IO指标、网络指标阈值
- 基于进程组的聚合判断（处理同名多进程情况）
- 前台窗口状态判断
- 可配置的检查间隔（默认10秒）
- 闲置持续时间超过设定阈值（默认30分钟）时进行进程优雅释放
- 支持两种闲置检测模式：累积计时和滑动窗口
- 支持加权滑动窗口，更准确地反映进程的当前状态
- 前台窗口活跃优先处理，确保用户正在使用的进程不会被误判为闲置
- 每日生成一个日志文件，最多保留7天
- 支持打包为exe文件，包含自启动脚本
- 提供用户配置文件，允许自定义设置
- 加强了异常处理和容错机制，提高程序稳定性
- 优化了性能，减少了资源消耗

## 配置说明

### 进程配置

进程配置信息通过Flask模拟服务端API提供，默认配置包含以下进程：

- msedge.exe：CPU阈值5.0%，内存阈值1500MB，IO阈值10MB/s，网络阈值1.0MB/s，累积计时模式
- bilibili.exe：CPU阈值3.0%，内存阈值500MB，IO阈值5.0MB/s，网络阈值0.5MB/s，普通滑动窗口模式
- EXCEL.EXE：CPU阈值2.0%，内存阈值800MB，IO阈值8.0MB/s，网络阈值0.1MB/s，普通滑动窗口模式
- MATLAB.exe：CPU阈值5.0%，内存阈值1000.0MB，IO阈值10.0MB/s，网络阈值1.0MB/s，加权滑动窗口模式

### 闲置检测模式配置

每个进程可以单独配置闲置检测模式，支持以下选项：

- `idle_detection_mode`：闲置检测模式，可选值：
  - `cumulative`：累积计时模式（默认）
  - `sliding_window`：滑动窗口模式
- `sliding_window_size`：滑动窗口大小，默认180（单位：检查次数）
- `sliding_window_idle_percentage`：闲置百分比阈值，默认90（单位：%）
- `sliding_window_weighted`：是否使用加权滑动窗口，默认false

### 用户配置

用户可以通过修改`user_config.json`文件来自定义设置：

- `api_url`：API地址，默认http://127.0.0.1:5000/api/process-configs
- `cache_duration`：配置缓存时间，单位秒，默认300
- `check_interval`：检查间隔，单位秒，默认10
- `log_level`：日志级别，可选DEBUG, INFO, WARNING, ERROR, CRITICAL，默认INFO
- `log_keep_days`：日志保留天数，默认7
- `max_processes_per_check`：每次检查的最大进程数，默认10
- `process_cache_expiry`：进程信息缓存时间，单位秒，默认5

## 加权滑动窗口应用流程

### 原理说明

加权滑动窗口是一种时间序列数据分析方法，用于更准确地检测进程的闲置状态。它的核心原理是：

1. **窗口滑动**：维护一个固定大小的队列，存储最近的进程状态
2. **加权计算**：对最近的状态赋予更高的权重，更重视当前状态
3. **闲置判定**：当窗口中闲置状态的加权比例达到阈值时，判定为闲置

### 应用流程

```
┌────────────────────────────────────────────────────────────┐
│                      监控循环                               │
└───────────────┬───────────────────────────────────────────┘
                │
                ▼
┌────────────────────────────────────────────────────────────┐
│        检测进程状态 (活跃/闲置/前台)                         │
└───────────────┬───────────────────────────────────────────┘
                │
                ▼
┌────────────────────────────────────────────────────────────┐
│     将状态添加到滑动窗口 (状态, 前台标志)                    │
└───────────────┬───────────────────────────────────────────┘
                │
                ▼
┌────────────────────────────────────────────────────────────┐
│      检查窗口是否包含前台活跃状态                           │
└───────────────┬───────────────────────────────────────────┘
                │
        ┌───────┴───────┐
        │               │
        ▼               ▼
┌────────────┐     ┌────────────┐
│  有前台活跃 │     │  无前台活跃  │
└────┬───────┘     └────┬───────┘
     │                 │
     ▼                 ▼
┌────────────┐     ┌────────────┐
│  判定活跃  │     │  计算加权闲置比例 │
└────────────┘     └────┬───────┘
                        │
                        ▼
             ┌───────────────────┐
             │  检查是否达到阈值  │
             └─────────┬─────────┘
                       │
               ┌───────┴───────┐
               │               │
               ▼               ▼
       ┌────────────┐     ┌────────────┐
       │  继续监控  │     │  终止进程  │
       └────────────┘     └────────────┘
```

### 权重计算方法

| 窗口位置 | 权重计算 | 示例（窗口大小=5） |
|---------|---------|------------------|
| 最早的状态 | (i+1)/window_size | i=0 → 0.2 |
| 较早的状态 | (i+1)/window_size | i=1 → 0.4 |
| 中间的状态 | (i+1)/window_size | i=2 → 0.6 |
| 较新的状态 | (i+1)/window_size | i=3 → 0.8 |
| 最新的状态 | (i+1)/window_size | i=4 → 1.0 |

### 优势

1. **更准确的闲置判定**：最近的状态权重更高，更能反映进程的当前状态
2. **更快的响应速度**：当进程状态变化时，能够更快地调整判定结果
3. **更好的适应性**：能够适应不同进程的特性，尤其是那些状态变化较快的进程
4. **前台窗口优先**：确保用户正在使用的进程不会被误判为闲置
5. **资源友好**：虽然计算复杂度略高，但内存消耗仍然很小

## 性能优化

1. **进程信息缓存**：使用缓存减少进程遍历频率
2. **配置缓存**：减少API调用频率，提高响应速度
3. **限制进程检查数量**：每次检查最多处理10个进程，避免资源消耗过大
4. **优化IO和网络监控**：减少采样时间，提高性能
5. **异常处理**：加强异常捕获和处理，提高程序稳定性
6. **滑动窗口优化**：只在窗口填满时进行计算，避免频繁计算
